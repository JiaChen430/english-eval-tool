# 项目管理方法论 - 事实单

最后更新：2026-02-26

## 核心决策

- [2026-02-26] 采用"分层记忆法" + Telegram Topics方式管理多项目
- [2026-02-26] 使用FACTS.md（导航页）+ docs/（详细文档）的分层存储
- [2026-02-26] 通过/checkpoint指令手动触发归档，不使用自动化
- [2026-02-26] 使用结构化Prompt约束保证逻辑一致性（方案1），暂不使用正则触发器和黑名单
- [2026-02-26] 增加跨topic共享信息机制（memory/topics/_shared.md）
  原因：不同topic需要共享通用规则、技术偏好、工作流程
  → 详见: AGENTS.md "Topic Memory"部分

## 当前架构

每个Telegram Topic对应一个项目，项目文件结构：
```
projects/{topic_name}/
├── FACTS.md          # 导航页（300-500 tokens）
├── docs/             # 详细文档（按需加载）
└── .history/         # 自动备份
```

上下文加载策略：
- 永远加载：FACTS.md（成本低）
- 按需加载：docs/下的详细文档

## 技术实现

- **平台**: Telegram Topics + OpenClaw
- **存储**: 本地Markdown文件系统
- **备份**: .history/文件夹（时间戳命名）
- **版本控制**: 暂不使用Git，等项目成熟后再考虑

## 质量与成本平衡原则

**核心理念**: 不为省token而牺牲清晰度

**具体实践**:
1. FACTS.md保持高信息密度（每句话都是核心事实）
2. 详细文档按需加载（不是每次都全部加载）
3. 用"导航 + 按需"替代"全量加载"
4. 用"覆盖式更新"替代"追加式堆积"
